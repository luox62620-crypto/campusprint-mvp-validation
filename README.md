# CampusPrint MVP Validation Repository

Repository URL: https://github.com/luox62620-crypto/campusprint-mvp-validation

This repository contains evidence and documentation for **CampusPrint**, a Software Start-Up Project by Team Splendor.

CampusPrint is a focused MVP concept for **no-queue residence/campus pickup printing for UTM KL students**. The current validation cycle tests whether students are willing to complete an online order-flow, show payment/reservation intent, and provide usability feedback before the team invests in a full automated system.

## MVP scope

The MVP is intentionally small. It focuses on a controlled **Google Form order-flow test** and does not include a full commercial printing operation.

### In scope

- Online order-flow test through Google Form
- Participant filtering for UTM KL students / residence or nearby students
- Simulated black-and-white print order details
- Pickup slot preference
- Payment or reservation intent
- Usability feedback and improvement comments

### Out of scope for this validation cycle

- 24/7 printing service
- Door-to-door delivery
- Color printing and binding
- Fully automated web/mobile platform
- Touch 'n Go API integration
- Real paid printing operation

## Main evidence results

- Completed responses: **12**
- Residence / nearby sample: **8/12 = 66.7%**
- Willing to submit real PDF online: **9/12 = 75.0%**
- Payment / reservation intent: **9/12 Yes = 75.0%; 12/12 Yes/Maybe = 100.0%**
- Mock reservation click intent: **11/12 = 91.7%**
- Would use during deadlines: **8/12 Yes; 12/12 Yes/Maybe**
- Average ease score: **3.75/5**

## Important note

This validation cycle used a **controlled MVP order-flow test**, not a full live paid printing service. The purpose was to collect early behavioral evidence before building a full automated product.
